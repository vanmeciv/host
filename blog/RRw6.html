This is a how-to for building Leaflet plugins using Github repositories.

Leaflet specifies that all plugins must be licensed. What is one of the licenses they recommend releasing plugin code under, and how do they suggest plugin developers let their users know about the pluginâ€™s licensing?
@yafred suggests "MIT License and BSD 2-Clause License are both good choices. You can either put it in the repo[sitory] as a `LICENSE` file or just link to the license from the Readme."

How does Leaflet know that developers have created new plugins and add them to its list of available plugins?
"Adding to the plugins list"
  • Fork the Leaflet repo.
  • In the `docs/plugins.md` file, find the section your plugin should go in, and add a table row with information and links about your plugin.
  • Commit the code to your fork.
  • Open a pull request from your fork to Leaflet's original repo.
